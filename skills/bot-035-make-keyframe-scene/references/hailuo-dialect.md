# Hailuo 02 first-last-frame dialect — the render engine, baked inline

Per-model mechanics for the **pinned-keyframe** render: the keyframe-chain identity lock, the
`--image` start / `end_image_url` hosted-end mapping, the silent-clips rule, the 6s/10s duration +
512P/768P resolution, and failure triage. Baked **inline** here because the runtime sandbox has
**no KB access** — this file IS the source of truth at runtime. New-architecture split: the K+1
keyframes are generated by the **shared `.claude/skills/video-toolkit/scripts/gen-image.sh`** (the nano-banana-pro chain +
hosted-url capture); only the per-scene Hailuo morph is the bot-local `scripts/gen-keyframe-clips.sh`.

## What makes this engine different (load-bearing for the fleet)

> **Seedance (BOT-033 per-beat i2v / BOT-034 ref2video)** carries shots with native audio. **Hailuo
> here (BOT-035)** is the **precise first-last-frame** engine: each scene's clip is PINNED on BOTH
> ends — a START keyframe AND an END keyframe — and Hailuo **morphs start → end**. So the journey is
> a chain of pinned states, and the motion of each scene is exactly the interpolation between two
> frames you control.

That is the headline: **pinned start AND end per scene**. You are not hoping the model lands the
ending — you hand it the ending. The trade is that Hailuo clips are **silent** (an ambient bed is
added at assembly, `--roomtone always`) and the per-scene transition is a morph, not a free-running
shot. Disclose this honestly in `summary.md`; never present the ambient bed as native audio.

## The journey model — K+1 states, K scenes

The `keyframe-plan.md` defines **K+1 pinned states** (state 0 … state K) and **K motion prompts**
(one per scene). Scene `i` morphs **state[i] → state[i+1]**. With 5 states you get 4 scenes; with
6 states, 5 scenes. The episode length ≈ `K × SCENE_DURATION` (default `K × 6s`).

## The engine — per state a keyframe, per scene a morph

### (1) The keyframes — nano-banana-pro via the SHARED gen-image.sh, chained for identity

Generate ALL K+1 keyframes first, with the shared driver. The identity lock is two-fold:
- the **FROZEN TOKENS** from the token seed kit (woven verbatim into every keyframe prompt — never
  paraphrase a locked token; "amber eyes" stays "amber eyes"), AND
- **`--ref <state[i-1] local png>`** for every state `i > 0`, so the SAME character literally
  carries forward from the previous keyframe. State 0 has no ref (it establishes the character).

```bash
# state 0 (no ref — establishes the character)
.claude/skills/video-toolkit/scripts/gen-image.sh work/state-00.txt artifacts/<slug>/keyframes state-00.png \
  --chain "fal-ai/nano-banana-pro,openai/gpt-image-2,fal-ai/nano-banana-2" \
  --aspect-ratio <16:9|9:16|1:1> --seed <seed> --max-cost 80

# state i>0 (ref = the PREVIOUS state's local png)
.claude/skills/video-toolkit/scripts/gen-image.sh work/state-NN.txt artifacts/<slug>/keyframes state-NN.png \
  --chain "fal-ai/nano-banana-pro,openai/gpt-image-2,fal-ai/nano-banana-2" \
  --aspect-ratio <AR> --seed <seed> --ref artifacts/<slug>/keyframes/state-PP.png --max-cost 80
```

- **NO `--resolution` flag.** nano-banana-pro REJECTS `--resolution` (exit non-zero) and the whole
  image chain falls through, skipping the primary. `gen-image.sh` accepts the flag for
  forward-compat but NEVER forwards it. Pass only `--aspect-ratio` (+ `--ref` + `--seed`).
- **`gen-image.sh` prints `model<TAB>local-path<TAB>hosted-url`. Capture BOTH:** the local path is
  the START upload for Hailuo; the **hosted url is the END-FRAME contract** for Hailuo. files[]
  entries are OBJECTS and the driver already does the python3 parse — you just read the three TAB
  fields.
- **Chain fallback:** `fal-ai/nano-banana-pro → openai/gpt-image-2 → fal-ai/nano-banana-2` — all
  three are reference-capable + aspect-capable, so the identity lock survives a fallback.
- A keyframe that produces a file but **no hosted url** can still be a scene's START (uploaded via
  `--image`) but **cannot be an END frame**. Record it; the per-scene logic falls back to a
  still-segment when a scene's END frame has no url.

### (2) The morph — Hailuo 02 image-to-video via the BOT-LOCAL gen-keyframe-clips.sh

```bash
RESOLUTION=768P HAILUO_MAX_COST=200 \
  scripts/gen-keyframe-clips.sh \
    work/scene-NN.txt \
    artifacts/<slug>/keyframes/state-NN.png \
    "<state[i+1] HOSTED url>" \
    6 \
    artifacts/<slug>/work-scenes/scene-NN.mp4
```

Under the hood the script runs:

```bash
ai-gen video "<motion prompt i>" \
  -m fal-ai/minimax/hailuo-02/standard/image-to-video \
  --image "<state[i] local png>" \
  "end_image_url=<state[i+1] HOSTED url>" \
  "duration=6" --resolution 768P --max-cost 200 --format json
```

- **`--image` = the START keyframe** (uploaded; a local path is fine — ai-gen uploads it). Hailuo
  treats it as the first frame.
- **`end_image_url=` MUST be the HOSTED url of the END keyframe** (`state[i+1]`). This is the whole
  point of first-last control — the clip MORPHS start → end. A local path will NOT work here; it
  must be the hosted url captured from the END keyframe's `gen-image.sh` run. Pass it as a
  `key=value` params pass-through (a positional `end_image_url=...` arg), NOT a `--flag`.
- **Slug:** `fal-ai/minimax/hailuo-02/standard/image-to-video` (the `standard` tier). The `pro`
  tier swaps `/standard/` for `/pro/` at a higher cost (env `HAILUO_MODEL`).
- **Duration:** Hailuo 02 takes **6s or 10s** reliably. Passed as `duration=6` in the params
  pass-through. If the model rejects the pass-through, the script retries WITHOUT it (the clip runs
  at the model default — disclose in summary.md).
- **Resolution:** `--resolution 768P` (or `512P` to trim cost). Hailuo DOES accept a resolution
  token here (unlike nano-banana-pro on the image side — do not confuse the two).
- **NO native audio.** Hailuo i2v clips come back **SILENT**. Audio is added at assembly
  (`assemble.sh --roomtone always`).
- **`motion prompt` shape:** describe the MOTION between the two pinned frames — one camera move +
  one subject action. Keep the subject large in frame; avoid the very darkest lighting. The two
  keyframes already fix the look — the motion prompt only steers the interpolation.

## Skip-and-continue (precise control degrades gracefully, never silently)

A scene whose Hailuo morph fails — or whose START local / END hosted url is missing — is **NOT
dropped from the timeline**. Build a **still-segment fallback** from the two boundary keyframes via
`scripts/still-segment.sh` so the journey stays K scenes long:
- **both keyframes present** → a `SCENE_DURATION`-second segment that holds state[i], then
  cross-fades (xfade 1s) to state[i+1] — a static "morph stand-in".
- **only one keyframe present** → hold that single still for `SCENE_DURATION` seconds.
- **neither present** → the scene is dropped (recorded).

The run is a clean recorded failure (no MP4) **only if EVERY scene fails** (no Hailuo clip and no
still fallback at any boundary). Every still-segment fallback is disclosed per-scene in `summary.md`.

## Assembly — shared assemble.sh: uniform-normalize → concat → ambient bed (ALWAYS)

The bot does NOT own an assembler — it calls `.claude/skills/video-toolkit/scripts/assemble.sh`:
`--pad-color black --roomtone always --verify summed --tol 2`. Because Hailuo clips are silent,
`--roomtone always` adds a quiet brown-noise room tone (≈−38dB) derived from the plan `Audio:` line.
It is an **added ambient bed, NOT native Hailuo audio** — say so in summary.md. The summed±2s gate
absorbs per-clip rounding across many short scenes.

## Cost

- Per scene = the START keyframe image cost (nano-banana-pro) + the Hailuo i2v cost. The END
  keyframe of scene `i` is the START keyframe of scene `i+1`, so **K+1 keyframes total, not 2K**.
  Use `.claude/skills/video-toolkit/scripts/cost.sh estimate …` for each, and `ai-gen balance` deltas to confirm —
  **never** the JSON `credits_used` field (it over-reports ~8× on i2v). 1 credit ≈ $0.004.
- A failed generation is not charged, but a needless re-render is — prefer fixing the composed
  prompt over re-rendering. Drop `--resolution` to `512P` or keep the Hailuo `standard` tier to
  trim cost while iterating.
