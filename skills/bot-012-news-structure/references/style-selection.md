# Style selection — which look to recommend

Set `recommended_style` from **what the story leads with** × **target platform**. The user can always override. Full table: `research/domain-analysis.md` §4 / `research/domain-model-fit.md` §2.

| Style | Recommend when the NewsDoc has… | Native platform / AR |
|---|---|---|
| **headline-highlight** | a strong `quote` OR a crisp `headline` + 1–2 `key_phrases` to annotate | LinkedIn / X / blog header — 1:1 or 16:9 |
| **breaking-news** | a time-sensitive `What` + a clear `source.name`; official / market / government | news feeds / broadcast — 16:9 or 1:1 |
| **kinetic-typography** | a `primary_stat` or a stat-led `headline`; short-form social | TikTok / Reels / Shorts — 9:16 |
| **minimal-editorial** | a clean `headline` + `dek`; premium / brand-calm; or nothing else clearly fits (SAFE DEFAULT) | brand channels / premium IG — any, often 16:9 or 1:1 |

## Decision shortcut
1. Is there a quotable line? → **headline-highlight**.
2. Is it urgent / official / time-stamped? → **breaking-news**.
3. Does a single number carry it, or is it for short-form social? → **kinetic-typography**.
4. Otherwise (premium/brand-calm, or unclear) → **minimal-editorial**.

If the user named a platform, default to its native aspect ratio (9:16 social, 16:9 desktop/YouTube, 1:1 feed) and let that nudge the style. Always state the one-line reason in `01-newsdoc.md`.
