# hf-voiceover — narration audio + word-level timing

## Purpose
Turn the beat-structured `02-script.md` into the project's **spoken audio** (`assets/vo/*.wav`) and the
**word-level timing** (`04-timing.json`) that the build and render phases need to sync captions and pace
beats. Both come through the keyless **`ai-gen`** SL8 proxy — TTS via Kokoro, then ASR (Wizper) over the
TTS output to recover per-word start/end times. This skill never authors HTML and never renders video; it
only produces the audio + timing assets that `hf-build` (caption blocks, beat clip timing) and `hf-render`
(audio mux) consume.

`$SKILL` = this skill's directory. Confirmed commands, voices, and the `-o-is-a-directory` gotcha live in
`references/ai-gen-audio.md`.

## When to use
- **Voiceover** (phase 4): after `hf-script` wrote `02-script.md`, before `hf-build`.
- **Re-voice** (JTBD-4): a follow-up like "use a different voice" / "re-voice with af_nova" — re-run from
  the SAME `02-script.md` (the narration text must NOT change; only the voice changes). Facts only change
  by re-running `hf-script` (phase 2).
- Do NOT use to write the script (that is `hf-script`), to build the composition (`hf-build`), or to
  produce cutouts/captures (`hf-assets`).

## Inputs
- `artifacts/<project-name>/02-script.md` (required) — the per-beat script. Each beat has a VO line; those
  lines, in order, are the narration. If it is missing/empty, record the failure in `state.md` and stop
  (run `hf-script` first) — do NOT invent narration.
- `voice` (optional) — resolve from the user request → else `context.md` (the onboarding voice default) →
  else `am_michael`. State which you used.
- `model` (optional) — default `fal-ai/kokoro/american-english`.

## Instructions

### 1. Extract the beat narration from the script
Read `02-script.md` and pull the **VO / narration line for each beat in order** (ignore the on-screen-text
and direction lines). Build a small list of `(beat_id, narration_text)` — `beat_id` is a stable kebab slug
(`beat-01`, `beat-02`, … or the script's own beat labels). This is the only place narration comes from;
never add or paraphrase sentences the script doesn't contain.

If the script has a single continuous narration with no explicit beats, treat each sentence (or each
storyboard scene, if `03-storyboard.md` exists) as a beat.

### 2. Synthesize the voiceover (TTS, per beat)
For each beat, synthesize a wav with the `tts.sh` wrapper. **`-o` is a DIRECTORY**, so the script targets a
per-beat temp dir and captures the produced wav via `ls`:
```bash
mkdir -p artifacts/<project-name>/assets/vo
bash "$SKILL/scripts/tts.sh" "<narration_text>" artifacts/<project-name>/assets/vo "<beat_id>" "<voice>" "<model>"
#                              ^text             ^out-dir                              ^stem    ^voice  ^model(optional)
```
`tts.sh` runs `ai-gen audio tts "<text>" -m <model> -o <tmpdir>` into a fresh temp directory, finds the
single produced wav there (`ls <tmpdir>/*.wav`), and moves it to
`artifacts/<project-name>/assets/vo/<beat_id>.wav`. It prints the final path and exits non-zero only if no
wav was produced. The Kokoro voice is selected by passing the voice id (see `references/ai-gen-audio.md` for
how the voice maps onto the model/`--voice` argument). Do all beats; then concatenate them in order into
`assets/vo/narration.wav` (the wrapper appends each clip's duration so the timing offsets line up) — or call
`tts.sh ... concat` once at the end. **Do not** call `ai-gen` directly; always go through `tts.sh` so the
directory-capture and the running-offset bookkeeping stay correct.

### 3. Recover word-level timing (ASR over the TTS wav)
For each beat's wav, run the `words.sh` wrapper, which calls `ai-gen audio stt "<wav>" -m fal-ai/wizper`,
parses the JSON for per-word `start`/`end` (when the model returns them), and writes the merged result to
`04-timing.json`:
```bash
bash "$SKILL/scripts/words.sh" artifacts/<project-name>/assets/vo artifacts/<project-name>/04-timing.json
#                               ^vo-dir (all beat wavs)            ^timing output
```
`words.sh` iterates every `<beat_id>.wav` in the vo dir (in filename order), transcribes each, and emits:
```json
{
  "beats": [
    { "beat": "beat-01", "wav": "assets/vo/beat-01.wav", "duration": 2.84,
      "words": [ { "text": "Ship", "start": 0.00, "end": 0.31 }, { "text": "faster", "start": 0.31, "end": 0.78 } ] }
  ],
  "words": [ { "text": "Ship", "start": 0.00, "end": 0.31, "beat": "beat-01" } ],
  "total_duration": 12.40,
  "source": "ai-gen audio stt fal-ai/wizper"
}
```
The per-beat `words[]` are **offset by the cumulative start of that beat** in `words.sh`, so the flat
`words[]` track is absolute timeline time (what caption blocks read). If Wizper returns only the transcript
text with no per-word timestamps, `words.sh` **falls back to even distribution**: it splits the transcript
into words and spreads them across the beat's measured `ffprobe` duration (it records
`"timing_method": "even"` in that beat so downstream knows the timings are approximate). It never fabricates
words that aren't in the transcript.

### 4. Headless fallback (TTS unreachable → render silent)
If `tts.sh` cannot produce a wav for a beat (ai-gen TTS unreachable / `success:false`), do NOT fail the
project and do NOT prompt the user:
- Skip that beat's audio, and still write a `04-timing.json` entry for it using the **script-estimated**
  pacing (≈ `max(0.8, words/2.5)` seconds per beat, the legibility pacing rule) with
  `"timing_method": "estimated"` and `"wav": null`.
- Record in `state.md` (phase 4) that the voiceover fell back to **silent** (which beats, why), so
  `hf-render` knows there is no audio stream to mux and the final summary reports it.
- This is the JTBD-1 documented fallback: "Kokoro/voice unavailable → render silent + note it" — the video
  still ships; the timing track still drives beat pacing.

### 5. Report
State the resolved **voice** and **model** used, how many beats got real TTS audio vs fell back to silent,
the total narration duration, and whether word timings are real (Wizper word-level) or estimated/even. Mark
`state.md` phase 4 done (or `blocked`/fallback-noted), and remember.

## Outputs
- `artifacts/<project-name>/assets/vo/<beat>.wav` — one narration wav per beat (+ `narration.wav`, the
  concatenated full track), generated by `ai-gen` TTS. (Absent for any beat that fell back to silent.)
- `artifacts/<project-name>/04-timing.json` — word-level timings: a per-beat `beats[]` with `words[]` and a
  flat absolute-time `words[]` track for caption sync, plus `total_duration` and the timing method.

## Examples

### Example 1: narrate a 12s teaser (JTBD-1)
`02-script.md` has 5 beats. Run `tts.sh` per beat into `assets/vo/`, then `words.sh` over the dir →
`04-timing.json` with `beats[]` and a flat `words[]`. Report: "voice `am_michael` (default), Kokoro;
5/5 beats voiced; 12.4s total; word-level timings from Wizper." → hand off to `hf-build`.

### Example 2: re-voice an existing project (JTBD-4)
User: "re-voice it with af_nova." Re-read the UNCHANGED `02-script.md`, set `voice=af_nova`, re-run steps
2–3 (overwriting `assets/vo/*.wav` + `04-timing.json`). Append a dated note to `state.md` ("re-voiced
am_michael → af_nova; narration text unchanged"). Do NOT touch the script facts.

### Example 3: TTS unreachable (headless fallback)
`tts.sh` returns `success:false` for every beat. Write `04-timing.json` from script-estimated pacing
(`timing_method:"estimated"`, `wav:null`), record "voiceover → silent fallback (ai-gen TTS unreachable)" in
`state.md`, and report it. `hf-build`/`hf-render` proceed without an audio stream.

## Troubleshooting
- **No wav after TTS / "where did the file go?"** — `-o` is a **directory**, not a filename. The wav lands
  at `<dir>/american-english-<ts>.wav`. `tts.sh` already captures it via `ls <dir>/*.wav` and renames it to
  `<beat_id>.wav`; if you called `ai-gen` directly you missed this — use `tts.sh`.
- **`success:false` in the ai-gen JSON** — the model is unreachable or the call was malformed. `tts.sh`
  exits non-zero and prints the JSON; take the silent fallback (step 4). Do not retry-loop forever.
- **`04-timing.json` has words but no timestamps** — Wizper returned text only; `words.sh` then used even
  distribution over the `ffprobe` duration (`timing_method:"even"`). Captions will be approximately synced —
  acceptable; note it.
- **Wrong voice / accent** — pass the right Kokoro voice id (see `references/ai-gen-audio.md`); the default
  model is the American-English Kokoro. A different language needs a different Kokoro model — say so, don't
  silently mistranslate.
- **`ffprobe`/`jq` absent** — `words.sh` falls back to `node` for JSON and a fixed nominal duration; timings
  become estimated. Prefer running in-sandbox where both are present.

## Quality Criteria
- [ ] One non-empty `assets/vo/<beat>.wav` per beat that TTS reached (or a recorded silent fallback for the rest).
- [ ] `04-timing.json` exists with a non-empty `words[]` track (real Wizper timings, or even/estimated with the method recorded).
- [ ] Narration text is exactly the script's VO lines — nothing invented, nothing dropped.
- [ ] On a re-voice, the script facts are unchanged; only the voice differs.
- [ ] The resolved voice/model and any silent/even/estimated fallback are reported and written to `state.md`.
- [ ] Keyless via `ai-gen`; no HeyGen auth/TTS, no cloud — never prompts the user when headless.
